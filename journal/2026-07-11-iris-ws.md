# 2026-07-11 — PRV10IRIS5: boot confiável (ex-IT5391, 192.168.15.189)

## Feito
- Pedro relatou: (A) PC sem POST até reseat da RAM; (B) boot "Ubuntu" do GRUB falhava, só Advanced funcionava.
- **Root cause de (B) confirmada por log**: entrada topo do GRUB bootava o kernel mais novo **6.17.0-14** — instalado órfão (sem headers, sem meta-pacote), com módulos NVIDIA em instalação **quebrada** (`iF`/`iU` no dpkg). Xorg: `Failed to load module "nvidia" (module does not exist)` → fallback nouveau → tela preta. O `GRUB_DEFAULT` já estava pinado no 6.14.0-36 (caminho Advanced OK).
- **Fix**: purge de `linux-image/modules-6.17.0-14` + 3 pacotes nvidia-6.17 + 2 metas quebradas (`linux-modules-nvidia-550/580-generic-hwe-24.04`); `dpkg --configure -a` limpo; GRUB regenerado — topo agora aponta 6.14.0-36 (validado: symlinks /boot e grub.cfg), fallback 6.8.0-49. dpkg sem half-installed.
- **Triagem de (A) no-POST**: causa provável = mau contato pós-manuseio (instalação das NICs). Riscos identificados: RAM genérica 2×8GB DDR4-2666 "BANSHEE" (manufacturer 0x0000), BIOS Gigabyte **F5 07/2023** (antiga, memory training), slots corretos (1/canal), EDAC/MCE sem erros.
- Reboot anterior da máquina já tinha validado persistência: hostname PRV10IRIS5, renames eth_*, MTU 9000, driver ngbe in-tree sem erros.

## Decisões
- Menos kernels = menos armadilhas: 6.17 removido em vez de consertar NVIDIA nele; 6.14.0-36 (HWE) é o kernel de produção, 6.8.0-49 de fallback.
- Sintoma (A) tratado como hardware: pendências físicas abaixo, sem mexer remoto.

## Adendo — pendrive de flash abortado
- Pendrive 3.8G (BEA4-7D07) preparado p/ Q-Flash: **NAND degradada detectada** — dados antigos com leituras divergentes a cada leitura (arquivo 1 GB: 4 hashes diferentes; sem erro no kernel = corrupção silenciosa). Gravações frescas liam estáveis.
- Backup best-effort em `~/backups/pendrive-BEA4-7D07-20260711/` + `LEIA-ME-INTEGRIDADE.txt`: **confiáveis** = 3× GuardLogix `.ACD`, mapas/, brochures; **não confiáveis** = 2 instaladores TwinCAT (rebaixar da Beckhoff), 1 PDF Laserline, arquivos CS 1.6.
- Decisão do Pedro: **abortar uso desse pendrive** p/ flash; conteúdo restaurado no stick; BIOS files removidos. F17 segue staged em `~/DRIVERS/bios-f17/` na PRV10IRIS5 + zip no CDN (sha256 `7675…23ec`) — preparar outro pendrive quando houver.
- Técnica útil: `dd iflag=direct | sha256sum` 2× p/ detectar flash silenciosamente instável (cache de página mascara o problema).
- GRUB_TIMEOUT=10 aplicado (menu 10 s; recordfail 30 s). Memtest: usar a entrada **sem** "serial console".
- **Boot direto "Ubuntu" validado pelo Pedro pós-purge** (6.14.0-36). Em seguida, `GRUB_DEFAULT=0` aplicado (pin por título removido — canônico Ubuntu; kernel novo via apt vira default já completo c/ NVIDIA). Verificado em grub.cfg: `set default="0"`.

## Adendo 3 — kiosk + power management (padrão IRIS-07)
- **Kiosk**: Google Chrome .deb (150.x, instalado offline via scp — WS sem internet no painel) + `/opt/kiosk/kiosk.sh` (loop watchdog, `xset -dpms`, flags do spec IRIS-07) + autostart GNOME (`~/.config/autostart/kiosk.desktop`). **URL configurável em `/etc/kiosk-url`** (1ª linha); sem o arquivo → placeholder `/opt/kiosk/placeholder.html`. Testado ao vivo na sessão X (:0): chrome --kiosk rodando.
- **Autologin GDM já estava ativo** (strokmatic, X11/WaylandEnable=false) — item do spec pré-existente.
- **Sem suspend**: `systemctl mask sleep/suspend/hibernate/hybrid-sleep.target` + gsettings via bus da sessão (idle-delay 0, sleep-inactive-ac/battery nothing, lock-enabled false). Gotcha: sem dbus-launch na WS — usar `DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/1001/bus`.
- **Pendente físico**: BIOS "AC BACK = Always On" (Settings→Platform Power) p/ boot automático ao energizar — fazer na mesma visita do flash F17.
- **Hardening (camadas 2+3)**: usuário **`kiosk`** (sem sudo, sem senha de login; autologin GDM trocado p/ ele; autostart migrado; `gnome-initial-setup-done` pré-criado) + dconf lockdown (overlay-key vazio, hot corners off, Alt+Tab/Alt+F2/Alt+F4/atalhos shell/terminal `[]`, notificações off, idle 0, sem sleep). `DontVTSwitch` **pulado de propósito** — Ctrl+Alt+F3 fica como acesso admin local c/ senha. Admin: SSH strokmatic (total); console = `pkill -u kiosk` → greeter → strokmatic c/ senha; break-glass = GRUB recovery.
- Gotchas: (a) `systemctl restart gdm3` NÃO derruba sessão gráfica existente — precisa `loginctl terminate-session N` antes; (b) **`kiosk` herdou uid 1000, que já era usado por processos de containers** (inference/redisinsight aparecem em `pgrep -u kiosk`) — só rótulo, sem impacto funcional, mas cuidado com `pkill -u kiosk` (pode matar processos de container! usar `pkill -f kiosk.sh` + `loginctl terminate-session`).
