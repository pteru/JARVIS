# 2026-07-09 — eip-stack (latência de comando e composição pylogix + strokmatic-eip)

## Feito

- **Orçamento de latência tela→motor** (jog 03007): frontend→Redis 10–50 ms;
  Redis→plc-monitor-eip é o vilão se for polling (mapa de produção ~70 tags ⇒
  batch pylogix 40–100 ms ⇒ captura 90–180 ms); CIP write 5–15 ms; scan+RPI
  20–80 ms; EtherCAT+drive <10 ms. Total: 40–90 ms bem desenhado, 100–300 ms
  descuidado. Crítico é a **parada**, não a partida: sobrecurso ≈ 15/45/90–120 mm
  a 50/150/300 mm/s ⇒ "rápido desloca, fino = lento + step 1 mm (imune)".
- **Regra 6 normativa no contrato C2 (v0.99-draft)**: comandos via Redis
  **Stream** (`XREAD BLOCK`) em thread dedicada + conexão CIP própria (pylogix
  não é thread-safe); **descarte por idade >300 ms** (crash-only acumula fila
  no restart); **stop prioritário**; mediana ≤150 ms / p95 ≤300 ms; eco
  `IF_Axis[n].LastCmdSeq` como medição permanente de round-trip.
- **pylogix × strokmatic-eip**: não são concorrentes. OpENer é stack de
  **dispositivo** (Classe 1 adapter + Classe 3 *servidor*) — não origina nada
  nem fala o simbólico Rockwell; pylogix é o originador Classe 3. §2.4 do
  contrato: Fase A (pylogix hot+cold) → Fase B (adapter 128 B @ RPI 10–20 ms no
  hot path; timeout de conexão = fail-safe §0.1 por hardware; pylogix fica no
  cold path). Alternativa "MSG invertido" registrada, não é o plano.
- **Issue de produto criada**: strokmatic/strokmatic-eip#4 — "EIP stack"
  composta (compose/pod), **nunca container único**: domínios de falha
  independentes (restart do poller crash-only não pode derrubar a conexão
  Classe 1), envelope certify-once intocado, redes distintas (Classe 1 exige
  host network/macvlan; Classe 3 roda em bridge), ciclos de vida opostos.

## Decisões

- Jog: `WSCfg_JogSpeed[3]` {50,150,300} mm/s editável + clamp por eixo;
  contínuo em **PV**, step em **PP relativo** (exato, não aproximado);
  discriminação clique×segurar é do frontend, nunca do PLC.
- IRIS 03007 = primeiro consumidor declarado da EIP stack (Fase A→B); contrato
  C2 agnóstico ao transporte — Fase B não muda nada do lado WS.
