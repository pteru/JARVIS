# 2026-06-11 — magna

## Feito

- Orientação inicial corrigida: o app é `sdk-inspection-grouping-**optimizer**`
  (não "analyzer").
- Criado projeto novo `Caixa de Roda (WH LH)` em `results/caixa-de-roda-wh-lh/`
  a partir de `references/caixa de roda`: 272 pontos de solda (W001–W272),
  modelo STEP tesselado/decimado (~100k faces), escala corrigida m→mm.
- 1ª rodada do optimizer (sem oclusão): 13 posições de câmera, 100% cobertura
  (272/272), câmera MV-SC6050M+16mm.
- Bug encontrado e corrigido no viewer: quando `project.json` não define
  `obstruction_file`/`stackups_file`, o dev-fallback injetava silenciosamente
  os dados do projeto Nissan — pontos exibidos não eram os do Caixa de Roda.
- Rodada com oclusão real (raycast contra a malha) + normais recalculadas a
  partir das superfícies (as normais do CSV original estavam erradas): 29
  posições, 255/272 (93.8%), 17 soldas bloqueadas.
- Investigação de acessibilidade (accessibility map, clearance distribution,
  accessibility-vs-ângulo via jobs em background) mostrou que o modelo de
  oclusão estava conceitualmente errado: soldas unem múltiplas chapas e devem
  poder ser inspecionadas de qualquer face.
- Implementado modelo "stackup-transparente": chapas dentro de um raio (~20mm,
  depois recalibrado para 10mm) do próprio weld não contam como oclusoras;
  apenas corpos além disso bloqueiam. Extraído também nº de chapas e
  espessura de cada uma por weld (`weld_sheets.json`).
- Resultado com stackup 10mm: 25 posições, 272/272 (100%), 0 pontos cegos.
- Corrigido bug de FOV: verificação usava o retângulo de FOV na distância de
  trabalho em vez da profundidade real da solda (`fov_at_distance`) — 16/443
  atribuições fora do frustum real → 0/391 após o fix.
- Avaliada câmera MV-SC6016M+16mm: sua distância mínima de trabalho (204mm)
  não alcançava 2 soldas (W064, W088) em paredes profundas, só visíveis em
  ângulo oblíquo (~12–15°). Implementada amostragem oblíqua de candidatos
  (`candidates.py::_cone_directions`, flag `--oblique`, default on).
- Resultado final com MV-SC6016M+16mm + amostragem oblíqua: 32 posições,
  272/272 (100%), 0 pontos cegos, 0/346 atribuições fora do frustum.

## Decisões

- Nome canônico do produto: `sdk-inspection-grouping-optimizer`.
- Modelo físico de oclusão adotado: stackup de chapas soldadas dentro de
  10mm de cada lado é transparente; só corpos além disso ocluem a
  linha de visada.
- Câmera final selecionada para o Caixa de Roda (WH LH): MV-SC6016M + 16mm,
  com amostragem oblíqua sempre ativa.
- Resultado pinado em `results/caixa-de-roda-wh-lh/` (`result.json`,
  `weld_sheets.json`, `welds.xlsx` corrigido); resultados de câmeras/estratégias
  anteriores mantidos como sidecar para comparação.
