# 2026-07-08 — blender

## Feito

- Levantado requisito: duas ferramentas de validação de visão de máquina para
  câmeras VisionKing body — mapa de cor por resolução efetiva (px/mm) e mapa
  de cor por ângulo entre visada da câmera e normal da superfície.
- Consolidado o diretório solto `sdk-blender-tools` (só um script de
  importação de calibração OpenCV, sem git) em um add-on Blender completo:
  `workspaces/strokmatic/sdk/sdk-blender-tools/strokmatic_vision_tools/`.
- Grillado o design via perguntas sequenciais: câmera configurada manualmente
  no Blender (não via JSON de calibração), escopo "objetos selecionados"
  (com opção "todos visíveis"), saída em vertex colors (sem bake de textura,
  sem subdivisão automática), modo duplo contínuo+threshold, single-camera
  (sem comparação multi-câmera na v1).
- Coletado o fluxo real de trabalho do usuário (câmeras perspective, STEP→
  FreeCAD→STL, render externo troca câmeras por modelos 3D com FOV
  translúcido, addon antigo de resolução custom é "bugado") e incorporado
  ao spec.
- Escritos e commitados spec (`docs/superpowers/specs/2026-04-13-blender-vision-tools-design.md`,
  incluindo toggle de unidade px/mm ↔ mm/px) e plano
  (`docs/superpowers/plans/2026-04-13-blender-vision-tools.md`, 10 tasks).
- Executado o plano via subagentes em paralelo (2 por vez): camera_utils,
  color_ramp, mesh_utils, calibration_importer, resolution_heatmap,
  angle_heatmap, ui_panel + testes — todas as 10 tasks completas e commitadas.
- Repo inicializado como git standalone (opção A — consistente com outros
  sdk-* repos) e pushado para `strokmatic/sdk-blender-tools`.
- Criado GitHub Release v1.0.0 com ZIP contendo só o add-on (sem
  tests/scripts), para instalação via "Install from Disk" sem clone/terminal.
- Escrito `scripts/release.sh` para versionamento futuro (bump de `bl_info`,
  build de ZIP, push, criação de release) — procedimento salvo no repo.
- Diagnosticado e corrigido bug reportado pelo colega: release e ZIP no
  GitHub ainda estavam com conteúdo/descrição antigos (pré-manifest);
  atualizados ambos.
- Iniciada instalação local de Blender 5.1.0 (via snap, já que Claude não
  pode rodar `sudo`) para reproduzir e validar o problema de instalação do
  colega em Windows.

## Decisões

- Blender mínimo suportado: 3.6 LTS, alvo de compatibilidade 3.6–4.2+, usando
  API `mesh.color_attributes` (não a legada `vertex_colors`).
- Unidade de heatmap de resolução é escolha do usuário (px/mm ou mm/px),
  com conversão recíproca e gradiente ajustado no display.
- Distribuição para o colega (Windows, sem git): GitHub Release com ZIP
  contendo apenas a pasta do add-on, atualizado via `release.sh` a cada
  versão.

## Pendências

- Validar Blender 5.1.0 localmente (instalação em andamento) para reproduzir
  o erro "addon não aparece na lista" relatado pelo colega e confirmar que o
  ZIP/manifest atualizado resolve em 5.x.
- Suite de testes Blender (fora de `color_ramp`) só roda dentro do Blender —
  ainda não executada de ponta a ponta.
- Multi-câmera (comparação/composite entre FOVs sobrepostos) ficou fora do
  escopo v1, cogitado para versão futura.
